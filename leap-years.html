Leap Years

The function

const leapYears = function(year) {
if (typeof year!=='number'){return 'ERROR';}
if (year <= 0){return 'ERROR';}
return (year%4 === 0 && year%100 !== 0) || year%400 === 0;
}

This was part of the Odin Project's introduction to test-driven development. I kept it around because, after a bunch of exercises where I really had to iterate my way through a bunch of fails and ended up with something pretty messy, I nailed the requirements on this one first time; always a nice feeling. Also I think this is the most efficient way of writing this function.1

The code

1 const leapYears = function(year) {
Line 1 is pretty straightforward. Declare the function and tell it to expect the year we're testing as an argument.

2 if (typeof year!=='number'){return 'ERROR';}
Line 2 checks the type of the input to make sure the year that's being tested is a number. If it's not, it throws an error. In production code, this would be more specific; the tests on this project were predefined to expect simply 'ERROR'

3 if (year <= 0){return 'ERROR';}
Line 3 checks the year is positive. If we wanted to get really pedantic here, this check should actually validate whether the year is greater than 1582 when the Gregorian calendar was adopted and we got the modern definition of leap years.2

4 return (year%4 === 0 && year%100 !== 0) || year%400 === 0;
And here's the real meat of the function. Something that would have looked like absolute nonsense to me not so long ago. In plain English:

  • Condition 1: year%4 === 0 Divide the year by 4 and check that there's no remainder. Leap years are every four years and fall on the years that are divisible by four.
  • Condition 2: year%100 !== 0 Divide the year by 100 and check that there is a remainder. Century years are not leap years. Unless...
  • Condition 3: year%400 === 0 Divide the year by 400 and check that there's no remainder. Every fourth century year is a leap year; 2000 was and 2400 will be.
  • Wrap all that up and put it together. Conditions 1 and 2 go hand in hand, if both are true then it's a leap year. Condition 3 is separate, so if both 1 and 2 are true or if 3 is true, it's a leap year.
  • return the result of that check; we get a boolean back telling us whether the tested year is or is not a leap year.
